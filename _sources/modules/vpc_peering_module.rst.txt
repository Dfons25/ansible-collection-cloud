.. Title

opentelekomcloud.cloud.vpc_peering -- Add/Update/Delete vpc peering connection from OpenTelekomCloud
++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

.. Collection note

.. note::
    This plugin is part of the `opentelekomcloud.cloud collection <https://galaxy.ansible.com/opentelekomcloud/cloud>`_ (version 0.0.1).

    To install it use: :code:`ansible-galaxy collection install opentelekomcloud.cloud`.

    To use it in a playbook, specify: :code:`opentelekomcloud.cloud.vpc_peering`.

.. version_added

.. versionadded:: 0.2.0 of opentelekomcloud.cloud

.. contents::
   :local:
   :depth: 1

.. Deprecated


Synopsis
--------

.. Description

- Add or Remove vpc peering from the OTC.


.. Aliases


.. Requirements

Requirements
------------
The below requirements are needed on the host that executes this module.

- openstacksdk
- openstacksdk >= 0.36.0
- otcextensions
- python >= 3.6

.. Notes

Notes
-----

.. note::
   - The standard OpenStack environment variables, such as ``OS_USERNAME`` may be used instead of providing explicit values.
   - Auth information is driven by openstacksdk, which means that values can come from a yaml config file in /etc/ansible/openstack.yaml, /etc/openstack/clouds.yaml or ~/.config/openstack/clouds.yaml, then from standard environment variables, then finally by explicit parameters in plays. More information can be found at https://docs.openstack.org/openstacksdk/

.. Seealso


.. Examples

Examples
--------

.. code-block:: yaml+jinja

    
    # Create a vpc peering.
    - opentelekomcloud.cloud.vpc_peering:
        name: "peering1"
        local_router: "local-router"
        project_id_local: "959db9b6017d4a1fa1c6fd17b6820f55"
        peer_router: "peer-router"
        project_id_peer: "959db9b6017d4a1fa1c6fd17b6820f55"

    # Change name of the vpc peering
    - opentelekomcloud.cloud.vpc_peering:
        name: "peering2"
        id: "959db9b6017d4a1fa1c6fd17b6820f55"

    # Delete a load balancer(and all its related resources)
    - opentelekomcloud.cloud.vpc_peering:
        name: "peering2"
        state: absent
